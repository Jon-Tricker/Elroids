Ship Components

The ship is made up of a number of 'component sets'. Each has a number of 'slots' which can contain a specific 'type' of component.

For now there is only one slot of each type.

Components can be bought and sold as described on the 'Ship Maintenance'.page.

Component types are:

Hulls

If the hull's status drops to 0% the ship is destroyed and it's GAME OVER.

Maximum speed is dependant on hull status. If pushed any harder ship would shake itself apart.

The hull also contains a 'template' defining the ship's other initial components.

Engines

Engines produce 'thrust'.

The amount of thrust is dependent on the engine's status.

The ship's acceleration is dependant on the amount of thrust and the ship's mass.

Deceleration is handled by the engines thrusting in the reverse of the current speed.

Weapons

Weapons may malfunction if status is < 50%.

Cargo bays

Each cargo bay has a maximum capacity. The capacity is reduced if the bay is damaged..

Dumping cargo

  • If capacity is reduced below the curent cargo load the least valuable cargo will be dumped.
  • If the bay is full and a more valuable cargo is loaded the less valuable is dumped.

If you have a problem with this policy debate it with the Wookie who does cargo management.

Avionics

All the electronic bits of the ship (Radar, compass, computers etc.).

Only one instance of radar or compass may be mounted. However you can carry a spare, in cargo, and swap out the mounted component if it becomed unservicable.

Component displays

In 'pilot' camera mode status displays for compnents are shown at the bottom of the screen.

This can cause a bit of clutter. So initially only the most useful components are shown.

Additional component displays can be added via the 'Display on/off' buttons in the "Ship Components/Upgrades" menu.
